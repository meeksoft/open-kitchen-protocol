# Contributing to Open Kitchen Protocol

Thank you for your interest in contributing to OKP!

## How to Contribute

### Reporting Issues
- Use GitHub Issues for bugs and feature requests
- Provide clear reproduction steps for bugs
- Include your use case for feature requests

### Proposing Changes

1. **Fork** the repository
2. **Create a branch** for your changes
3. **Make your changes** following our guidelines
4. **Submit a Pull Request** with a clear description

### Specification Changes

Changes to the OKP specification require:
- A clear rationale
- Backward compatibility analysis
- Example use cases
- Schema updates (if applicable)

### Guidelines

- Keep changes focused and minimal
- Update documentation alongside code
- Follow existing formatting conventions
- Add examples for new features

## Versioning

OKP follows semantic versioning:
- **Major** (2.0) - Breaking changes
- **Minor** (1.1) - New features, backward compatible
- **Patch** (1.0.1) - Bug fixes, clarifications

## Code of Conduct

Be respectful, constructive, and inclusive. We're building something together.

## Questions?

Open a GitHub Discussion for questions about the spec or implementation guidance.
